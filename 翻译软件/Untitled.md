Windows 划词翻译工具（Qwen）需求整理版
1. 项目定位与目标
开发一款基于 Windows 的划词翻译工具。用户在任意应用选中文本后，通过悬浮工具栏触发翻译/润色，后端调用阿里云百炼 Qwen 模型，提供高效、稳定的文本处理体验。

核心目标：

支持英汉互译、英文润色
支持自定义指令处理文本
支持历史记录与收藏管理
提供可持续使用的常驻翻译体验（不自动关闭弹窗）

2. 功能范围（分阶段）

P0（MVP 必须上线）
2.1 划词与触发
输入源：以剪贴板文本为唯一可靠输入（MVP）
触发方式：
划词后显示悬浮工具栏
兼容“自动复制后读取剪贴板”策略
支持至少一个全局快捷键

2.2 悬浮工具栏
默认显示 4 个按钮：
英译中
英文润色
中译英
复制
四个按钮均支持：
自定义指令
自定义顺序

2.3 翻译与润色能力（Qwen）
模式：
英→中
中→英
英文润色
自定义指令模式
调用阿里云百炼 Qwen 接口返回结果
错误处理覆盖：
超时
网络异常
鉴权失败
支持失败后快速重试

2.4 结果弹窗行为（重点）
弹窗内容：原文 + 结果
弹窗属性：
始终置顶
可拖拽
可调整大小
可固定位置
生命周期规则：
多次划词翻译过程中不重复弹新窗（复用同一窗口）
除非用户手动关闭，否则不自动关闭、不最小化到任务栏

2.5 历史与收藏
历史记录字段：
时间
原文
模式
结果
历史条数上限：100 条（超限按策略清理，见 5.3）
可查看最近历史
收藏功能：
历史项可加入/取消收藏
独立收藏列表

2.6 配置
API Key 配置
默认模式配置
快捷键配置
P1（体验增强）
自定义指令模板（Prompt Preset）
历史/收藏：搜索、筛选、删除
翻译结果多版本（如：更正式/更简洁）
开机启动
托盘常驻

P2（高级能力）
术语表/词汇记忆
多模型切换（不同 Qwen 模型）
云同步（历史与收藏）

3. 非功能需求
3.1 性能
从触发到结果展示目标：1~3 秒（网络良好）
3.2 稳定性
接口失败可重试
程序不因接口异常崩溃
3.3 安全与隐私
API Key 本地加密存储（Windows Credential Manager / DPAPI）
历史记录支持一键清空
3.4 可维护性
模块化架构，便于后续扩展 OCR、多模型、云同步

4. 技术方案约束（已确定）
桌面框架：Electron
全局能力：全局快捷键 + 剪贴板监听 + 系统托盘
本地存储：SQLite（历史/收藏）+ 配置文件
模型接入：阿里云百炼 OpenAI 兼容接口（Qwen）
UI 形态：简洁悬浮窗（输入、模式切换、结果、收藏）

5. 核心流程与规则
5.1 主流程
用户在任意应用中选中文本
弹出悬浮工具栏
用户点击功能按钮（英译中/润色/中译英/复制）
程序读取剪贴板文本（必要时执行复制动作）
按当前模式/自定义指令构造 prompt
调用 Qwen API
在固定结果窗展示结果
自动写入历史
用户可收藏/复制/再次处理
5.2 Prompt 输出约束
强约束模型仅返回目标文本，避免冗余说明（减少输出不稳定）
5.3 历史清理策略（建议落地）
达到 100 条后自动删除最旧记录（FIFO）
收藏项不随普通历史自动删除（建议单独管理）

6. 风险与应对
跨应用划词兼容性风险：不同软件选区读取差异大
应对：MVP 统一采用“复制+剪贴板”方案
快捷键冲突风险
应对：允许用户自定义快捷键
模型输出不一致风险
应对：模板化 prompt + 输出格式约束
网络波动风险
应对：超时、重试、用户友好提示

7. 已确认立项条件（冻结项）
框架：Electron
MVP 不做 OCR
历史上限：100 条
需要离线模式（仅查看历史/收藏）
需要中英文 UI
发布形式：msi 安装包
