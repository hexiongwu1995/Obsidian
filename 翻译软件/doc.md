# 划词翻译软件规格说明书

## 1. 项目概述

### 项目名称
**划词翻译器** (Word Translator)

### 项目类型
Windows 11 桌面应用程序 - 轻量级划词翻译工具

### 核心功能摘要
一款基于阿里云百炼 Qwen 模型的划词翻译工具，支持选中即译、英汉互译、英文润色功能，提供历史记录管理和收藏夹功能。

### 目标用户
Windows 11 中文用户，需要频繁进行中英文翻译和英文润色的场景。

---

## 2. 技术架构

### 技术栈
- **框架**: Electron 32.x
- **前端**: HTML/CSS/JavaScript + Vue 3 (轻量级 UI)
- **构建工具**: electron-builder
- **API**: 阿里云百炼 Qwen API (DashScope)
- **存储**: SQLite (better-sqlite3) + electron-store (设置)
- **加密**: Windows DPAPI + AES-256

### 模块划分

```
┌─────────────────────────────────────────────────────────┐
│                    Main Process                         │
├─────────────────────────────────────────────────────────┤
│  WindowManager    │  TranslationService  │  DataStore  │
│  - 主窗口管理       │  - API 调用            │  - SQLite   │
│  - 工具栏窗口       │  - 错误处理            │  - 加密     │
│  - 系统托盘         │  - 请求队列            │  - 历史记录 │
├─────────────────────────────────────────────────────────┤
│                    Renderer Process                     │
├─────────────────────────────────────────────────────────┤
│  TranslationUI   │  HistoryView  │  FavoritesView      │
│  - 对话式显示      │  - 列表浏览     │  - 文件夹管理       │
│  - 操作栏          │  - 分页加载     │  - 收藏操作         │
└─────────────────────────────────────────────────────────┘
```

## 4. 功能规格

### 4.1 核心翻译功能

#### 触发机制
1. 全局监听: 使用 Windows API 监听鼠标事件和剪贴板变化
2. 检测逻辑:
   - 监听全局鼠标左键释放事件
   - 检测剪贴板内容变化
   - 过滤短时间内的重复触发 (300ms 防抖)

#### 工具栏行为
- 显示时机: 检测到有效选中文本后 200ms 内弹出
- 显示位置: 鼠标指针右下方 (避免超出屏幕)
- 隐藏时机: 点击任意按钮或点击屏幕其他位置

#### 四个功能按钮
| 按钮 | 功能 | API Prompt |
|------|------|-------------|
| 英译中 | 将英文翻译为中文 | "请将以下英文翻译成中文：" |
| 润色 | 语法检查+自然重写 | "请检查并润色以下英文，使其更自然流畅：" |
| 中译英 | 将中文翻译为英文 | "请将以下中文翻译成英文：" |
| 复制 | 复制原文到剪贴板 | N/A |

### 4.2 翻译引擎

#### API 配置
- 服务商: 阿里云百炼 (DashScope)
- 模型: qwen-turbo (默认) 或 qwen-plus
- 端点: https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation

#### 请求格式
```javascript
{
  model: "qwen-turbo",
  input: {
    prompt: promptTemplate + selectedText
  },
  parameters: {
    result_format: "message",
    temperature: 0.3,
    max_tokens: 2000
  }
}
```

#### 错误处理
| 错误类型     | 用户提示                 |
| -------- | -------------------- |
| API 密钥无效 | "API 密钥无效，请检查设置中的密钥" |
| API 密钥过期 | "API 密钥已过期，请重新设置"    |
| 速率限制     | "请求过于频繁，请稍后再试"       |
| 网络断开     | "网络连接失败，请检查网络设置"     |
| 模型错误     | "翻译服务暂不可用，请稍后重试"     |

### 4.3 历史记录

#### 数据结构
```javascript
// History Item
{
  id: number,
  originalText: string,      // 原文
  translatedText: string,   // 译文
  mode: string,             // 'en2zh' | 'zh2en' | 'polish'
  createdAt: timestamp,      // 创建时间
  isFavorite: boolean       // 是否收藏
}

// Folder
{
  id: number,
  name: string,
  createdAt: timestamp
}
```

#### 加载策略
- 启动时加载最近 10 条记录
- 底部"加载更多"按钮每次加载 10 条
- 历史记录永不清空 (除非用户手动删除)

### 4.4 收藏夹系统

#### 功能
- 收藏单条翻译记录
- 创建/重命名/删除文件夹
- 将收藏移动到指定文件夹
- 从收藏夹视图访问

### 4.5 首次启动向导

#### 流程
1. 启动软件 → 显示欢迎界面
2. 引导用户输入 API 密钥
3. 测试密钥有效性
4. 成功 → 进入主界面
5. 失败 → 提示错误，允许重试

## 5. 数据流设计

### 5.1 核心数据流

```
用户选中文字
    ↓
Main Process 检测 (全局钩子)
    ↓
显示浮动工具栏
    ↓
用户点击功能按钮
    ↓
Renderer 发送消息给 Main Process
    ↓
Main Process 调用 TranslationService
    ↓
TranslationService 调用阿里云 API
    ↓
返回结果 → 显示在主窗口
    ↓
保存到历史记录 (SQLite)
```

### 5.2 数据存储

| 数据类型 | 存储方式 | 加密 |
|----------|----------|------|
| API 密钥 | Windows DPAPI | 是 |
| 历史记录 | SQLite | 否 |
| 收藏夹 | SQLite | 否 |
| 设置项 | electron-store | 否 |
| 提示词模板 | electron-store | 否 |
